## Parent

#1 拾流订阅中心 MVP 后端实现 PRD

## What to build

实现已登录状态下修改密码。

端到端贯穿 service + handler + 测试：
- 已鉴权的使用者提交旧密码 + 新密码；校验旧密码正确、新密码满足最少 12 字符，更新为新的 bcrypt(cost 12) 哈希。
- 旧密码错误或新密码不合规返回对应业务错误。
- 不提供 Web 端忘记密码流程，本切片不涉及。

## Acceptance criteria

- [ ] 已登录可用 旧密码 + 新密码 修改密码，新密码以 bcrypt cost 12 存储
- [ ] 旧密码错误返回业务错误，不更新
- [ ] 新密码 <12 字符返回校验错误，不更新
- [ ] 修改成功后用新密码可登录、旧密码失效
- [ ] service 层用 gomock 覆盖：成功、旧密码错误、新密码不合规
- [ ] handler 层受 JWT 中间件保护，未鉴权请求被拒
- [ ] `go build ./...`、`go test ./...`（相关包）通过
