#1 拾流订阅中心 MVP 后端实现 PRD

## What to build

实现内容条目上的处理状态切换、内容标记与音频播放进度持久化。

端到端贯穿 handler + service + 测试：
- 处理状态：手动把内容条目标记为已完成、也能从已完成改回未处理；阅读 / 收听进度不自动改变处理状态。
- 内容标记：稍后处理、收藏可叠加在处理状态之上，可独立设置 / 取消。
- 音频播放进度持久化：保存音频型内容条目的播放位置，供继续收听；不持久化文本阅读位置。

## Acceptance criteria

- [ ] 可手动在 未处理 ↔ 已完成 之间切换处理状态
- [ ] 稍后处理 / 收藏标记可独立设置与取消，且与处理状态叠加（互不覆盖）
- [ ] 音频播放进度可保存与读取；文本阅读位置不持久化
- [ ] 阅读 / 收听进度更新不改变处理状态
- [ ] service 用 gomock 覆盖状态切换、标记叠加、进度持久化
- [ ] handler 用 httpexpect 断言各操作响应
- [ ] `go build ./...`、`go test ./...`（相关包）通过
